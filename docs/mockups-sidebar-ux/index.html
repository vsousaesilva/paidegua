Quatro caminhos
para o pAIdegua
conviver com o PJe.

Hoje o painel lateral da extensão se sobrepõe à tela do PJe — quando aberto, cobre o conteúdo dos autos e do painel do usuário. Este dossiê compara, lado a lado, as quatro arquiteturas possíveis para que os dois ambientes ocupem a tela ao mesmo tempo. Cada caminho tem um mockup navegável e uma leitura honesta de custo e risco.

Os caminhos
00

Sobreposição

estado atual · linha de base

O sidebar desliza da direita por cima do PJe. É a forma como o pAIdegua opera hoje. Serve de referência para comparar com as três alternativas.

  • esforço nenhum
  • risco nenhum
  • UX cobre o PJe
Abrir mockup
01

Empurrar o PJe

mesma aba · CSS no host

Ao abrir, a extensão aplica margem à direita no documento do PJe e ocupa o espaço liberado. Tudo numa só aba — sem comunicação entre contextos.

  • esforço baixo
  • risco médio (layout)
  • UX lado a lado
Abrir mockup
02

Side Panel do Chrome

painel nativo do navegador

A extensão migra para a API chrome.sidePanel. O painel passa a ser desenhado pelo Chrome — o PJe não sabe que ele existe e nada no DOM do host muda.

  • esforço alto
  • risco baixo no PJe
  • UX ideal
Abrir mockup
03

Janela separada

popup desacoplado · multi-monitor

chrome.windows.create abre o pAIdegua em uma janela própria, que o usuário posiciona ao lado do PJe — ou em outro monitor. Total liberdade, mais custo de comunicação.

  • esforço médio
  • risco baixo
  • UX livre / desordenada

Recomendação

Para um experimento rápido, ir de 01 · Empurrar atrás de um toggle em “Mais opções”. Para o pAIdegua duradouro, planejar 02 · Side Panel como um projeto dedicado de UI v2.

O caminho 01 cabe num MVP comprovável em uma sprint e usa toda a base de código atual — com a ressalva clara de que pode quebrar telas específicas do PJe e exige validação por tela. O caminho 02 é a arquitetura tecnicamente correta para o longo prazo, mas exige refatorar a camada de UI: separar a árvore de componentes do DOM do host, mover o orquestrador para um sidepanel.html próprio e reescrever a comunicação detector ↔ sidebar via chrome.runtime.
