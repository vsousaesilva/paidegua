O que acontece

Painel desenhado pelo Chrome

A extensão registra um sidepanel.html via API chrome.sidePanel. O Chrome reserva uma faixa à direita da janela e renderiza nela fora do tab. O PJe não sabe que o painel existe — não há mexida em CSS do host.

Custo

v2 da camada de UI

O sidebar passa a viver em outro contexto que o content script. Toda comunicação detector ↔ sidebar (carregar documentos, OCR, abrir painel de audiência) vira chrome.runtime.sendMessage. Vários painéis montados dentro do PJe seguem como overlay; o sidebar fica enxuto.

Vantagem

Risco zero no PJe

Como o painel não convive com o DOM do host, nenhum layout do PJe é afetado — datatables, modais, gráficos seguem desenhando como sempre. O usuário ainda pode redimensionar a faixa via gesture nativa do Chrome.
